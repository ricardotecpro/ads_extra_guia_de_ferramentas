# Quiz: Aula 02 - Gestão de Projetos 📊

1.  **O que é o método Kanban?**
    *   ( ) Uma linguagem de programação orientada a objetos.
    *   (x) Um sistema visual de gestão de trabalho que utiliza cartões e colunas.
    *   ( ) Um tipo de banco de dados não relacional.
    *   ( ) Uma regra do Git para fazer commits.
    *   *Explicação: O Kanban foca na visualização do fluxo de trabalho e na limitação do trabalho em progresso.*

2.  **Qual o objetivo da coluna "Backlog" em um quadro ágil?**
    *   ( ) Guardar os arquivos que serão deletados.
    *   (x) Listar todas as ideias, tarefas e bugs que precisam ser feitos no futuro.
    *   ( ) Mostrar apenas o que está pronto.
    *   ( ) Colocar o código que está com erro.
    *   *Explicação: O Backlog é a "reserva" de trabalho que ainda não foi iniciada.*

3.  **No Trello, o que representa um "Card"?**
    *   ( ) Uma coluna inteira do projeto.
    *   ( ) O nome do usuário.
    *   (x) Uma tarefa individual ou item de trabalho.
    *   ( ) O histórico de logs do servidor.
    *   *Explicação: Cada card é uma unidade de trabalho que se move entre as listas.*

4.  **Qual a principal diferença entre Jira e Trello?**
    *   ( ) O Jira é apenas para designers.
    *   (x) O Jira é mais robusto e focado em metodologias ágeis complexas (Scrum/Kanban) para grandes times.
    *   ( ) O Trello é pago e o Jira é sempre gratuito.
    *   ( ) Não há diferença, são a mesma ferramenta.
    *   *Explicação: O Jira oferece relatórios avançados e fluxos de trabalho customizáveis para empresas.*

5.  **O que é uma "Sprint" no contexto de gestão de projetos?**
    *   ( ) Um comando para rodar o código mais rápido.
    *   (x) Um período de tempo fixo (geralmente 2 semanas) focado em entregar um conjunto de tarefas.
    *   ( ) O ato de deletar o banco de dados.
    *   ( ) Um tipo de animação em sites.
    *   *Explicação: Sprints ajudam a manter o foco e entregas constantes em ciclos curtos.*

6.  **Para que servem os "Labels" (Etiquetas) em um card?**
    *   ( ) Para deixar o quadro mais colorido apenas.
    *   (x) Para categorizar tarefas (ex: Bug, Feature, Prioridade Alta).
    *   ( ) Para esconder tarefas secretas.
    *   ( ) Para indicar o desenvolvedor que criou a tarefa.
    *   *Explicação: Etiquetas permitem filtrar e identificar rapidamente o tipo ou urgência de uma tarefa.*

7.  **No GitHub, as "Issues" servem para:**
    *   ( ) Hospedar o código fonte exclusivamente.
    *   (x) Rastrear bugs, sugestões e tarefas diretamente ligadas ao repositório de código.
    *   ( ) Fazer o deploy do site.
    *   ( ) Criar o visual do site (Design).
    *   *Explicação: Issues são o sistema de gestão de tarefas integrado ao ecossistema do GitHub.*

8.  **O que significa o termo "Escopo" de um projeto?**
    *   ( ) O tempo que o desenvolvedor gasta no café.
    *   (x) O conjunto de funcionalidades e limites definidos para o trabalho.
    *   ( ) A pasta onde o código é salvo.
    *   ( ) O nome da empresa que contratou o serviço.
    *   *Explicação: Definir o escopo evita que o projeto cresça infinitamente sem direção.*

9.  **O que é um "Blocker" (Impedimento)?**
    *   ( ) Uma ferramenta de segurança.
    *   (x) Algo que impede uma tarefa de avançar (ex: depender de outra pessoa ou erro técnico).
    *   ( ) Um tema escuro do Trello.
    *   ( ) O ato de desligar o computador.
    *   *Explicação: Identificar blockers é essencial para o gestor de projetos remover obstáculos do time.*

10. **Qual a vantagem de integrar o Slack com o Trello/Jira?**
    *   ( ) Para poder jogar games no chat.
    *   (x) Para receber notificações automáticas no chat quando uma tarefa muda de coluna.
    *   ( ) Para o Slack deletar tarefas automaticamente.
    *   ( ) Não existe essa integração.
    *   *Explicação: A integração mantém o time informado sem precisar abrir várias abas no navegador.*
